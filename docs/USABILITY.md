## Боли, которые закрыли (GIAB-демо)

| Было | Стало |
|------|--------|
| Ссылка `…-p4-i1.jpeg` → 404 или папка «Документы» | API без расширения, `Content-Disposition: inline` |
| Битый href `%3Ca%20class=` (HTML в URL) | Порядок автолинковки исправлен |
| «Открыть рисунок» уводит с контекста | Галерея + лайтбокс в диалоге и на «Главной» |
| Сырой Vision «Проанализируем…» | Librarian-аннотации и ранжирование фигур |
| Один чанк на документ | Несколько релевантных чанков + CLIP-рисунки |

## Сценарии приёмки (UX)

### GIAB — графики и таблицы воды

1. Демо → «Мультимодальный поиск» → вопрос про жёсткость / флотацию.
2. Ответ: текст + блок **«Рисунки из документа»** (не ссылки).
3. Клик по миниатюре → лайтбокс, Esc закрывает.
4. Вкладка «Главная» → те же рисунки в «Источниках», кнопка «Увеличить».
5. Нигде нет перехода на `localhost:8000/api/media/…` в новой вкладке.

### Обычный диалог

1. Follow-up «свести в таблицу» → таблица читаема в пузыре.
2. F5 → история на месте; граф восстанавливается.
3. Режим «Без LLM» → badge понятен, ответ без «галлюцинаций».

## Анти-пatterns (не возвращать)

- Ссылки на внутренние `/api/media/images/…` как основной способ просмотра.
- Markdown-ссылки, которые превращаются во вложенный HTML в href.
- Сырой вывод Vision в ответ пользователю.
- Требовать chunk_id / doc_id от пользователя в вопросе.
- Блокирующая индексация CLIP при старте API (>30 с без UI-feedback).
